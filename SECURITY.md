# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please:

1. **Do NOT open a public GitHub issue**
2. Email us at: security@your-org.com
3. Include a description of the vulnerability
4. Include steps to reproduce
5. Include potential impact

### What to expect:

- **Acknowledgment**: Within 48 hours
- **Status Update**: Within 5 business days
- **Resolution Timeline**: Based on severity
  - Critical: 7 days
  - High: 14 days
  - Medium: 30 days
  - Low: 90 days

## Security Practices

This project uses:
- Automated SAST scanning via Semgrep
- Dependency vulnerability scanning via Trivy
- Secret detection via Gitleaks
- Container image scanning
